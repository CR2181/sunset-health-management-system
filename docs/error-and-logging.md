# 错误处理与日志规范

## 目标

错误处理和日志的目标是让系统问题能定位、业务问题能追踪、安全问题能审计。

MVP 阶段至少做到：

- 参数错误能明确提示。
- 登录失败不泄露敏感信息。
- 设备事件、告警处理、护理任务变更有记录。
- 后端异常有日志。
- 前端错误不导致页面白屏。

## 错误处理规则

后端优先使用 NestJS 的异常能力：

```text
BadRequestException
UnauthorizedException
ForbiddenException
NotFoundException
ConflictException
InternalServerErrorException
```

禁止用普通字符串随意抛错。  
禁止吞掉异常后假装成功。  
禁止所有错误都返回 200。

## 错误分类

```text
参数错误：请求字段缺失、格式不合法。
认证错误：未登录、token 过期、账号密码错误。
权限错误：角色不允许访问。
业务错误：任务已完成、告警已处理、设备未绑定。
外部错误：设备网关、AI 服务、数据库、第三方接口异常。
系统错误：代码异常、配置缺失、服务不可用。
```

## 日志级别

```text
debug：开发调试信息，生产默认关闭。
info：服务启动、健康检查、关键业务成功。
warn：可恢复异常，如设备离线、AI 推理超时。
error：请求失败、数据库异常、外部服务异常。
fatal：服务无法启动、数据库不可连接、核心配置缺失。
```

## 必须记录的业务日志

- 用户登录成功和失败。
- 老人档案新增、修改、删除。
- 护理任务创建、完成、超时、复核。
- 告警创建、确认、处理、关闭。
- 设备上线、离线、低电量、故障。
- AI 事件接收、去重、生成告警。
- 家属查看敏感信息。

## 日志字段

建议日志至少包含：

```json
{
  "timestamp": "2026-06-14T10:20:00+08:00",
  "level": "info",
  "requestId": "req_xxx",
  "userId": "user_xxx",
  "module": "alerts",
  "action": "alert.resolve",
  "message": "Alert resolved",
  "metadata": {}
}
```

## 审计日志

审计日志和普通运行日志不同。审计日志用于回答：

```text
谁在什么时间看了什么数据？
谁改了老人档案？
谁处理了告警？
谁导出了敏感信息？
```

审计日志后续应单独建表，例如：

```text
audit_logs
```

## 前端错误处理

前端请求 API 失败时：

- 不白屏。
- 显示友好提示。
- 控制台记录技术错误。
- 不展示 token、密码、数据库连接信息。

当前前端保留了后端不可用时的演示数据兜底。正式 MVP 验收前，应减少对本地假数据的依赖。

## 设备和 AI 错误

设备和 AI 服务异常不能直接拖垮管理后台。

推荐策略：

```text
视频预览失败 -> 显示摄像头离线
AI 推理失败 -> 保留人工告警入口
设备上报失败 -> 写入重试队列
重复事件 -> 规则引擎去重
```

## 隐私要求

日志中禁止记录：

- 明文密码。
- JWT token。
- 完整身份证号。
- 家属手机号全文。
- RTSP 账号密码。
- 私密区域视频地址。
