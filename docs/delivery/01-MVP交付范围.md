# 第一代 MVP 交付范围

## 试点目标

在一个养老院、一个试点区域内，用 30 床虚构数据验证以下最小闭环：

```text
老人档案 -> 护理任务 -> 安全告警 -> 人工处置 -> 设备状态 -> 审计追溯
```

## 本次交付

- 六类账号登录、退出和刷新恢复：超级管理员、院长、护理员、康复师、家属、访客。
- 前端菜单、页面、路由和后端接口的角色权限控制。
- 老人档案、护理任务、实时告警、历史告警、设备台账、摄像头台账和审计日志。
- 护理任务开始、完成；告警确认、解决、误报；设备心跳和 AI 模拟事件复核。
- MySQL/MariaDB 或 PostgreSQL 数据库存储。
- 30 位虚构老人及关联任务、告警、设备、摄像头和账号。
- 部署、备份、验收、培训、隐私和演示资料。

## 不在本次范围

- 真实 AI 推理、医疗诊断或自动护理决策。
- 真实 RTSP 转码播放、MQTT、ONVIF、GB28181 正式设备网关。
- HIS、医保、长护险生产对接。
- 家属独立 App、短信和微信通知。
- 多院区、多租户、自动计费和生产级高可用集群。

## 数据口径

- 正式业务列表只从后端 API 和数据库读取。
- 后端不可用时前端显示错误，不使用本地静态业务数据伪装正常。
- 访客只看虚构演示内容；家属只看绑定老人；护理员只看责任老人。
- 当前数据库可能保留试点前已有的虚构记录，不自动删除。

## 完成判定

必须同时满足：构建通过、自动测试通过、匿名读取返回 401、越权返回 403、30 床数据存在、写操作有审计日志、部署和备份步骤可执行。
