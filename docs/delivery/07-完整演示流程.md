# 15 分钟完整试点演示流程

## 演示前准备

```powershell
cd backend
npm run service:start
npm run smoke
```

确认健康检查正常，浏览器使用本机地址，避免展示 `.env`、数据库密码和真实个人信息。

## 0:00–1:00 登录与角色

1. 使用超级管理员登录。
2. 指出菜单由统一权限配置生成，手动输入无权限 URL 仍由后端拦截。
3. 强调试点账号只用于本机演示。

## 1:00–3:00 工作台

1. 展示授权老人数、待处理任务、当前告警和在线设备。
2. 说明指标来自数据库，不是页面硬编码。
3. 展示一个告警区域和一个设备状态。

## 3:00–5:00 老人档案

1. 进入“老人档案”，确认 URL、标题和菜单高亮同步。
2. 展示床位、护理等级、风险和虚构健康摘要。
3. 使用家属账号说明只能看到 `RES-001`。

## 5:00–8:00 护理任务

1. 使用护理员账号进入“护理任务”。
2. 选择责任老人任务，点击开始处理。
3. 点击完成并说明状态写入数据库。
4. 说明其他老人任务即使知道 ID 也会返回 403。

## 8:00–11:00 告警闭环

1. 进入“安全告警”，确认一条告警。
2. 解决另一条告警，填写试点说明。
3. 标记一条误报。
4. 切换“告警记录”，展示实时处置与历史追溯的区别。

## 11:00–13:00 设备和 AI 预留

1. 进入“设备管理”，展示 30 台关联设备和 9 路摄像头台账。
2. 超级管理员模拟设备心跳。
3. 说明摄像头地址无效，当前不播放真实视频。
4. 模拟 AI 事件并人工复核，强调没有真实模型诊断。

## 13:00–14:00 审计追溯

1. 进入“审计日志”。
2. 找到登录、任务、告警、设备或 AI 操作。
3. 说明日志不包含密码和 JWT。

## 14:00–15:00 隐私隔离与收尾

1. 切换家属账号，展示单老人范围。
2. 切换访客账号，展示只能进入演示工作台。
3. 退出登录，确认返回登录页。
4. 明确下一阶段是设备联调、迁移脚本、HTTPS 和现场恢复演练。
