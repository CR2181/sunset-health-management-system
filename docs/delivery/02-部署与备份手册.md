# 部署与备份手册

## 适用环境

- Windows 10/11 或 Windows Server。
- Node.js 20 或更高版本。
- 当前项目自带便携 MariaDB；也支持外部 MySQL/PostgreSQL。
- 试点服务器建议至少 4 核 CPU、8 GB 内存、100 GB 可用磁盘。

## 首次部署

```powershell
cd C:\Users\Andy\Documents\养老院安全康复系统\backend
Copy-Item .env.example .env
npm ci
npm run build
npm run service:start
npm run smoke
```

打开：

```text
http://127.0.0.1:3000/src/index.html
```

## 环境变量

- `DB_TYPE`：`mysql` 或 `postgres`。
- `DB_HOST/DB_PORT/DB_USERNAME/DB_PASSWORD/DB_DATABASE`：数据库连接。
- `JWT_SECRET`：生产环境必须替换为高强度随机值。
- `JWT_EXPIRES_IN`：试点默认 8 小时。
- `CORS_ORIGINS`：只填写实际允许访问的管理端地址。
- `DB_SYNC`：本地试点可为 `true`；正式生产必须为 `false` 并使用迁移脚本。

`.env` 不得提交到 GitHub，不得通过聊天、邮件或截图传播。

## 健康检查

```powershell
Invoke-RestMethod http://127.0.0.1:3000/api/health
```

预期：`status=ok`、`database=up`。

## MariaDB 备份

```powershell
$backend = "C:\Users\Andy\Documents\养老院安全康复系统\backend"
$bin = Join-Path $backend ".runtime\mariadb\mariadb-11.4.12-winx64\bin"
$backupDir = Join-Path $backend "backups"
New-Item -ItemType Directory -Force $backupDir | Out-Null
$backup = Join-Path $backupDir ("sunset_health-" + (Get-Date -Format "yyyyMMdd-HHmmss") + ".sql")
& (Join-Path $bin "mysqldump.exe") -h 127.0.0.1 -P 3306 -u root -p --single-transaction --default-character-set=utf8mb4 --result-file=$backup sunset_health
```

命令会安全提示输入数据库密码。备份完成后检查文件大小大于 0，并复制到受控的加密存储。

## MariaDB 恢复演练

先停止 NestJS，保留数据库运行：

```powershell
npm run service:stop
npm run db:start
$mysql = "C:\Users\Andy\Documents\养老院安全康复系统\backend\.runtime\mariadb\mariadb-11.4.12-winx64\bin\mysql.exe"
& $mysql -h 127.0.0.1 -P 3306 -u root -p --default-character-set=utf8mb4 sunset_health --execute="source C:/完整路径/backups/sunset_health-日期.sql"
npm run service:start
npm run smoke
```

恢复必须先在非生产副本演练。不得直接覆盖唯一一份现场数据库。

## 回滚

1. 停止服务并完成数据库备份。
2. Git 切回已验收提交。
3. `npm ci`、`npm run build`。
4. 如数据库结构不兼容，恢复对应版本备份。
5. 启动并执行健康检查和冒烟测试。
