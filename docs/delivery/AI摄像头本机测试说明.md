# AI 摄像头本机测试说明

## 准备

1. 在 `backend` 目录执行 `npm run service:start`。
2. 使用 Chrome/Edge 打开 `http://127.0.0.1:3000/src/index.html`。
3. 使用管理员、院长或护士演示账号登录，进入“摄像头管理”。
4. 只在公共测试区域操作，不得对准卧室、卫生间或其他私密区域。

## 测试步骤

1. 点击“开启摄像头”，在浏览器权限提示中确认本次摄像头授权。
2. 页面显示本机画面和“mock 检测运行中”。普通帧不会产生风险事件。
3. 点击“模拟跌倒/离床/徘徊”之一。
4. 查看“最近事件”和“自动告警”状态，再进入“告警中心”。
5. 对告警执行确认、解决或误报，检查状态和审计日志。
6. 点击“停止摄像头”，确认画面停止。离开页面、退出登录或关闭页面也会自动释放。

## 预期结果

- 未点击开启前不请求摄像头权限。
- Mock 按钮产生 AI 事件；达到阈值时产生告警。
- 同一来源同类型 60 秒内更新同一活动告警。
- 处置告警同步关联 AI 事件。
- 不生成本地视频文件或证据帧，Git 状态不出现媒体文件。

## 故障提示

- 浏览器拒绝权限：页面提示失败，不自动重复申请；可在浏览器站点设置中重新授权。
- 非 localhost/HTTPS：本机摄像头不可用，改用 `127.0.0.1` 或 HTTPS。
- 本地 YOLO 不可用：若允许 fallback，页面显示降级到 mock；否则显示 AI 服务不可用。
- 后端不可用：写操作失败并显示错误，不伪造保存成功。

AI 辅助摘要仅供人工复核，不构成医疗诊断或护理指令。严禁提交真实老人、家属、员工、机构隐私数据和摄像头账号密码。
