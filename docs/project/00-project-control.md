# 00-项目总控与进度

## 作用

管理整个项目的总方向、当前阶段、下一步任务、风险和待办事项。

## 当前阶段

当前处于第一代 MVP 试点骨架阶段。

目标不是一次性做完整智慧养老平台，而是先跑通：

```text
管理端展示 -> 后端 API -> 数据库 -> 设备/AI 事件预留 -> 后续扩展
```

## 已完成

- 创建并推送 GitHub 公开仓库。
- 确定 MVP 技术路线。
- 安装并运行本机 MariaDB/MySQL 环境。
- 搭建 NestJS 后端。
- 管理端前端可访问后端 API。
- 新增 MVP 架构、接口、错误日志、开发规范文档。
- 后端整理为 `modules/*` 模块结构。
- 新增健康检查、统一返回、统一错误处理。
- 新增设备事件和 AI 事件预留接口。
- 创建 00-08 项目线程。

## 进行中

- 按 00-08 系列整理项目资料和后续任务。
- 固化 MVP 试点版本的项目管理文档。

## 缺失

- 每个系列的详细进度页。
- 数据库 ERD 和迁移策略。
- 前端页面拆分计划。
- GitHub 版本发布记录。
- 周报模板。

## 下一步

2. 检查 README 与项目索引是否一致。

## 当前风险

- 前端仍是单文件原型，后续维护会变难。
- 数据库当前依赖 `DB_SYNC=true`，生产环境不能这样使用。
- 设备和 AI 目前只是预留接口，不能当成真实能力。
- 侧边栏排序无法真正永久固定，永久顺序以 `docs/project/README.md` 为准。

## 试点产品化更新

项目正在从演示系统升级为第一代可试点产品。当前后端聚焦最小闭环：

```text
老人档案 -> 护理任务状态 -> 告警确认/解决 -> 设备心跳 -> AI事件复核 -> 审计日志
```

当前仍是 MVP：不做真实 AI 推理、不做真实视频流、不做 MQTT 网关、不做家属 App、不做医院/HIS 深度接入。
